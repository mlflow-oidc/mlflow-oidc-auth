**Create `web-react/src/features/workspaces/components/bulk-assign-modal.tsx`:**

Props:
```typescript
interface BulkAssignModalProps {
  isOpen: boolean;
  onClose: () => void;
  onGrant: (name: string, permission: PermissionLevel) => Promise<boolean>;
  onSuccess: () => void;
  title: string; // "Bulk Assign Users" or "Bulk Assign Groups"
  nameLabel: string; // "Usernames" or "Group Names"
  namePlaceholder: string; // "user1, user2, user3" or "group1\ngroup2\ngroup3"
}
```

Imports: `Modal`, `Button`, `PermissionLevelSelect`, `useToast`, `PermissionLevel` type.

State:
- `namesInput` (string) — the textarea content
- `permission` (PermissionLevel) — default "READ"
- `isSubmitting` (boolean)
- `results` (Array<{ name: string; success: boolean }> | null) — shown after processing

**UI layout:**
1. `Modal` wrapper with title prop and `width="lg"`
2. `<textarea>` for entering names (comma-separated or newline-separated)
   - Label: `{nameLabel}*`
   - Placeholder: `{namePlaceholder}`
   - Rows: 4
   - Styling: standard form textarea matching the codebase's input patterns: `w-full px-3 py-2 border rounded-md focus:outline-none text-ui-text dark:text-ui-text-dark bg-ui-bg dark:bg-ui-bg-dark border-ui-border dark:border-ui-border-dark focus:border-btn-primary dark:focus:border-btn-primary-dark transition duration-150 ease-in-out`
   - Helper text below: "Enter names separated by commas or newlines"
3. `PermissionLevelSelect` for choosing permission level
4. Cancel (variant="ghost") and "Assign All" (variant="primary") buttons
5. If `results` is not null, show a summary section:
   - Success count: "✓ {N} assigned successfully"
   - Failure count (if any): "✗ {M} failed: {comma-separated failed names}"

**Submit logic:**
1. Parse `namesInput`: split by `,` and `\n`, trim each, filter out empty strings, deduplicate
2. If empty after parsing, show toast "No names provided" and return
3. Set `isSubmitting = true`
4. For each name, call `onGrant(name, permission)` sequentially (not in parallel — to avoid overwhelming the server and to properly track individual results)
5. Collect results as `{ name, success: boolean }`
6. Set `results` with the collected results
7. Count successes and failures
8. If all succeeded: toast `"Successfully assigned {N} permissions"` (success), call `onSuccess()`, call `onClose()`
9. If some failed: toast `"{successes} assigned, {failures} failed"` (warning/error), keep modal open showing results, call `onSuccess()` to refresh the list
10. Set `isSubmitting = false`

Use `key={isOpen ? "open" : "closed"}` on the form container div for reset when modal reopens.

Reset `results` to null when `namesInput` changes (so the results summary disappears when user edits input).
